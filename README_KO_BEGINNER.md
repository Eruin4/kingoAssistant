# kingoAssistant 초보자용 설명서

이 문서는 발표자료를 만드는 팀원이 프로젝트를 빠르게 이해할 수 있도록 쓴 쉬운 설명서입니다.
개발자용 명령어보다 "무엇을 만들었고, 왜 만들었고, 어떻게 동작하는지"를 중심으로 설명합니다.

## 한 줄 소개

kingoAssistant는 안드로이드에서 음성으로 일정을 말하면, 서버가 음성을 글자로 바꾸고 AI가 내용을 해석해서 일정이나 할 일을 정리해 주는 앱입니다.

예를 들어 사용자가 이렇게 말할 수 있습니다.

```text
내일 오후 3시에 회의 추가해줘
선형대수 문제풀이 할 일로 추가해줘
다음 주 월요일 과제 일정 추가해줘
```

앱은 바로 일정을 저장하지 않고, 먼저 "이렇게 추가할까요?"처럼 제안으로 보여줍니다.
사용자가 수락하면 일정이나 할 일 목록에 반영됩니다.

## 만든 이유

일정 앱은 직접 날짜와 시간을 입력해야 해서 번거롭습니다.
이 프로젝트는 사용자가 말로 편하게 입력하면 앱과 서버가 다음 일을 대신 처리하도록 만든 것입니다.

- 음성 녹음
- 음성을 텍스트로 변환
- 문장의 의미 분석
- 일정과 할 일 구분
- 추가 또는 삭제 제안
- 사용자가 확인한 뒤 저장

즉, "음성 비서 + 일정 관리 앱"을 직접 구현한 프로젝트입니다.

## 전체 구조

프로젝트는 크게 두 부분으로 나뉩니다.

```text
안드로이드 앱  ->  HomeVoice 서버  ->  Whisper / KingoGPT
```

각 역할은 다음과 같습니다.

- 안드로이드 앱: 사용자가 보는 화면입니다. 음성을 녹음하고, 일정과 할 일을 보여줍니다.
- HomeVoice 서버: 앱에서 받은 요청을 처리합니다. 음성 파일을 받고, AI 결과를 저장합니다.
- Whisper: 음성 파일을 글자로 바꾸는 STT 엔진입니다.
- KingoGPT: 글자로 바뀐 문장을 읽고 "일정 추가", "할 일 추가", "삭제", "질문" 같은 구조화된 명령으로 바꿉니다.

STT는 Speech To Text의 줄임말입니다.
쉽게 말하면 "음성을 글자로 바꾸는 기능"입니다.

## 사용 흐름

1. 사용자가 안드로이드 앱에서 마이크 버튼을 누릅니다.
2. 앱이 사용자의 말을 WAV 음성 파일로 녹음합니다.
3. 앱이 서버로 음성 파일을 보냅니다.
4. 서버가 Whisper로 음성을 텍스트로 변환합니다.
5. 서버가 변환된 텍스트와 현재 일정 정보를 KingoGPT에 보냅니다.
6. KingoGPT가 사용자의 의도를 JSON 명령으로 바꿉니다.
7. 서버가 그 명령을 바로 실행하지 않고 제안 목록에 저장합니다.
8. 앱이 제안을 화면에 보여줍니다.
9. 사용자가 수락하면 일정 또는 할 일에 반영됩니다.

## 왜 바로 저장하지 않고 제안으로 보여주나요?

AI가 사용자의 말을 항상 완벽하게 이해하는 것은 아닙니다.
예를 들어 "내일 과제"라고 말했을 때 실제로는 할 일일 수도 있고 일정일 수도 있습니다.

그래서 이 앱은 AI가 판단한 내용을 먼저 사용자에게 보여줍니다.
사용자가 확인하고 수락해야 실제 일정에 저장됩니다.
이 방식은 실수로 잘못된 일정이 저장되는 문제를 줄입니다.

## 앱 화면 구성

안드로이드 앱에는 네 가지 주요 화면이 있습니다.

- Chat: 사용자의 명령과 AI 응답을 대화처럼 보여줍니다.
- Calendar: 일정 목록을 보여줍니다.
- Tasks: 할 일 목록을 보여줍니다.
- Settings: 서버 주소, API 키, 알림 시간, TTS 설정을 바꿀 수 있습니다.

추가 기능도 있습니다.

- 마이크 버튼으로 음성 입력
- 홈 화면 위젯에서 바로 녹음
- 왼쪽으로 밀어서 일정 또는 할 일 삭제
- 일정 알림
- AI 답변을 음성으로 읽어 주는 TTS

TTS는 Text To Speech의 줄임말입니다.
쉽게 말하면 "글자를 음성으로 읽어 주는 기능"입니다.

## 서버가 저장하는 데이터

서버는 일정 데이터를 작은 JSON 파일로 저장합니다.

저장되는 주요 정보는 다음과 같습니다.

- calendar_1: 첫 번째 일정표
- calendar_2: 두 번째 일정표
- tasks: 할 일 목록
- proposals: 아직 사용자가 수락하지 않은 제안 목록
- chat_history: 최근 대화 기록

이 프로젝트는 큰 데이터베이스 대신 JSON 파일을 사용합니다.
프로젝트 규모가 작고 일정 데이터 구조가 단순하기 때문에 관리하기 쉽습니다.

## 서버 API란 무엇인가요?

API는 앱과 서버가 서로 대화하기 위한 약속입니다.
안드로이드 앱은 버튼을 누르거나 음성을 보내야 할 때 서버의 특정 주소로 요청을 보냅니다.

대표적인 API는 다음과 같습니다.

```text
GET  /health          서버가 살아 있는지 확인
GET  /schedule        현재 일정 가져오기
POST /command         텍스트 명령 보내기
POST /stt-command     음성 파일 보내기
POST /proposal/accept 제안 수락하기
DELETE /task/{id}     할 일 삭제하기
DELETE /event/{id}    일정 삭제하기
```

현재 운영 서버 주소는 다음과 같습니다.

```text
https://eruin.mooo.com/voice
```

## 실제 KingoGPT 솔버 위치

이 저장소에 예전에 있던 `kingogpt_api_solver.py`는 참고용 사본이었습니다.
실제 운영 서버는 별도로 서버에 있는 KingoGPT 프로젝트를 사용합니다.

운영 서버 기준 실제 경로는 다음과 같습니다.

```text
/home/eruin/kingoGPT/kingogpt_api_solver.py
```

Docker 컨테이너 안에서는 다음 경로로 보입니다.

```text
/app/kingoGPT/kingogpt_api_solver.py
```

즉, 이 프로젝트는 HomeVoice 앱과 서버를 담당하고, KingoGPT 솔버는 외부 프로젝트를 연결해서 사용합니다.

## 발표 때 강조하면 좋은 점

이 프로젝트에서 강조할 만한 부분은 다음과 같습니다.

- 안드로이드 앱과 서버를 직접 연결했습니다.
- 음성 인식, AI 분석, 일정 저장 흐름을 하나로 만들었습니다.
- AI 결과를 바로 저장하지 않고 사용자 확인 단계를 넣었습니다.
- 일정과 할 일을 구분하도록 프롬프트와 보조 로직을 만들었습니다.
- 서버를 Docker로 배포해서 운영 중인 주소에서 사용할 수 있게 했습니다.
- API 키를 사용해서 아무나 서버를 호출하지 못하게 했습니다.

## 발표용 쉬운 설명 예시

아래 문장은 발표자료에 그대로 옮겨도 됩니다.

```text
저희 프로젝트는 음성으로 일정을 관리하는 안드로이드 앱입니다.
사용자가 앱에서 "내일 오후 3시에 회의 추가해줘"라고 말하면,
앱은 음성 파일을 서버로 전송합니다.
서버는 Whisper를 이용해 음성을 텍스트로 바꾸고,
KingoGPT를 이용해 그 문장이 일정 추가인지, 할 일 추가인지, 삭제 요청인지 판단합니다.
AI가 만든 결과는 바로 저장되지 않고 제안으로 표시되며,
사용자가 수락해야 실제 일정에 반영됩니다.
이렇게 해서 음성 입력의 편리함과 사용자 확인의 안정성을 함께 가져가도록 설계했습니다.
```

## 시연 순서 추천

1. 앱을 열고 Settings 화면에서 서버 주소가 들어 있는지 보여줍니다.
2. 마이크 버튼을 누르고 "내일 오후 3시에 회의 추가해줘"라고 말합니다.
3. 서버 응답 후 제안이 뜨는 것을 보여줍니다.
4. 제안을 수락합니다.
5. Calendar 화면에서 일정이 추가된 것을 보여줍니다.
6. "선형대수 문제풀이 할 일로 추가해줘"를 입력하거나 말합니다.
7. Tasks 화면에서 할 일이 추가된 것을 보여줍니다.
8. 항목을 왼쪽으로 밀어 삭제하는 기능을 보여줍니다.

## 주의할 점

- API 키는 발표자료나 GitHub에 공개하면 안 됩니다.
- 서버의 `.env` 파일은 비밀 설정 파일입니다.
- KingoGPT 토큰 파일도 공개하면 안 됩니다.
- 음성 녹음 파일, 로그, 빌드 결과물은 Git에 올리지 않습니다.
